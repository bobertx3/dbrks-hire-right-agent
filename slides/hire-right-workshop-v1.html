Step 1 · Notebook 00

Ingest — Bronze Layer

Raw candidate data lands in Delta Lake as a Bronze table — schema inferred from CSV, resumes uploaded as PDFs to a Unity Catalog Volume.

  • 20 HR candidates across 4 open roles (Director HR, VP TA, Dir Comp&Ben, CPO)
  • read_files() for schema-on-read from CSV — no ETL boilerplate
  • Resume PDFs ingested to /Volumes/bx4/hrd_2030/hr_resumes/
  • Audit column ingested_at stamped on every row

Step 3 · Notebooks 03–04

Build for AI — Gold Layer + Genie Space

Gold tables aggregate candidate scores for analytics and ML training. Genie Space gives the agent natural language access to HR data. Vector Search enables semantic resume retrieval.

  • Gold table joins candidates + job requirements with computed hiring metrics
  • Business Semantics layer: COMMENT ON TABLE + column descriptions make Genie accurate
  • Genie Space configured with trusted assets and sample questions
  • Vector Search index synced from resume chunk Delta table — 1536-dim embeddings

Step 4 · Notebook 05

Train ML Model — Hire / No-Hire Prediction

MLflow autolog captures every experiment run. Best model (GradientBoostingClassifier) selected by AUC-ROC and registered to Unity Catalog with the @champion alias.

  • 8 features: education, experience, leadership, certification, skills_match, industry_relevance, interview, culture_fit
  • GradientBoostingClassifier — 100 estimators, lr=0.1, max_depth=3
  • MLflow autolog: input examples, model signatures, metrics
  • UC registration: bx4.hrd_2030.hr_predictive_hiring

Step 5 · Notebook 05

Register & Serve — @champion Model Endpoint

Best run registered to Unity Catalog Model Registry and aliased as @champion. A serverless Model Serving endpoint exposes it as a REST API — the agent calls it via predict_hiring_score.

  • UC model name: bx4.hrd_2030.hr_predictive_hiring
  • Endpoint: hr-predictive-hiring-endpoint
  • Scale-to-zero serverless compute — no idle costs
  • Agent tool calls endpoint via Databricks SDK token-auth

How Databricks agents connect, communicate, and stay governed — across every framework, orchestrator, and enterprise system.

Databricks agents participate as peer agents in multi-agent workflows. Any orchestrator can delegate multi-turn tasks and receive governed, traceable responses.

→ MLflow Tracing → Evals & quality scores → SQL-queryable logs

Agents, models, skills, and tools are Unity Catalog objects — with RBAC, lineage, audit logs, and access policies applied automatically across every call.

→ RBAC & ABAC → Full audit trail → Lineage across agents
Step 6 · Notebook 06

Build & Deploy Agent — ResponsesAgent + MLflow 3.0

The Hire Right Agent is a ResponsesAgent (OpenAI-compatible ChatAgent) packaged with MLflow 3.0, evaluated with mlflow.genai.evaluate(), and deployed as a serverless endpoint.

  • 4 tools declared as resources: Genie, Vector Search, ML Endpoint, Mailgun
  • Evaluation: Safety, RelevanceToQuery, custom jnj_tone scorer
  • agents.deploy() — scale-to-zero, env vars injected at serve time
  • Registered as bx4.hrd_2030.hire_right_agent in UC
