MLflow 3.0

ResponsesAgent — The New Agent Standard

MLflow 3.0 introduced ResponsesAgent — an OpenAI Responses API-compatible interface that turns any framework or model into a first-class, governable Databricks service.

🔌

Drop-in OpenAI Replacement

Any client built for OpenAI works with your Databricks agent — just swap base_url. No SDK changes, no prompt rewrites.

🏗️

Wrap Any Framework

LangChain, LlamaIndex, LangGraph, custom Python — wrap it in ResponsesAgent and deploy through the same governed interface.

📊

MLflow Tracing + UC Governance

Every agent call is tracked end-to-end with MLflow spans, versioned in the model registry, and subject to Unity Catalog permissions — full enterprise governance on agentic workloads.

⚖️

Quality Gates via mlflow.genai.evaluate()

Built-in LLM judges score safety, relevance, and custom criteria before deployment. Results flow to Delta tables — queryable with SQL.

④ · Notebook 05

Train ML Model — Hire / No-Hire Prediction

MLflow autolog captures every experiment run. Best model (GradientBoostingClassifier) selected by AUC-ROC and registered to Unity Catalog with the @champion alias.

MLflow Experiments · ml-hr-predictive-hiring · GradientBoosting selected

Register · Notebook 05

Register & Serve — @champion Model Endpoint

Best run registered to Unity Catalog Model Registry and aliased as @champion. A serverless Model Serving endpoint exposes it as a REST API — the agent calls it via predict_hiring_score.

Model Serving · hire-right-agent-endpoint & hr-predictive-hiring-endpoint

⑤ · Notebook 06

Build & Deploy Agent — ResponsesAgent + MLflow 3.0

The Hire Right Agent is a ResponsesAgent (OpenAI-compatible ChatAgent) packaged with MLflow 3.0, evaluated with mlflow.genai.evaluate(), and deployed as a serverless endpoint.

MLflow Evaluate · JnJ Tone judge · agent quality assessment
