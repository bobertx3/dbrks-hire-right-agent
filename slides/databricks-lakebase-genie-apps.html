Lakebase — Data Layer

Operational data.
On your lakehouse.

Lakebase brings full transactional database capabilities — row-level operations, low-latency reads/writes, ACID guarantees — to the same open Delta Lake format your analytical workloads already use.

  • No ETL pipelines moving data between OLTP and OLAP systems
  • Open formats — no proprietary lock-in, works with Spark, dbt, and your BI tools
  • Governed by Unity Catalog — one permission model for all data
  • Scales from millisecond lookups to petabyte analytics on the same table

Lakebase — project dashboard · branch management · point-in-time restore

04 / 09

AI/BI Genie — Intelligence Layer

Ask a question.
Get a real answer.

Genie is a natural language interface to your structured data. Business users ask questions in plain English — Genie writes the SQL, executes it against your Unity Catalog tables, and explains what it found.

  • Works on any Unity Catalog table — no data movement or preparation
  • Verified SQL: admins curate trusted answers that override LLM guesses
  • Learns from follow-up — multi-turn conversations refine results
  • Embeddable via REST API — Genie becomes a tool inside any agent or application
  • Governed access — users only see data their permissions allow

Genie Space — natural language → verified SQL → visualization

05 / 09

Databricks Apps — Application Layer

Ship AI apps.
Without the plumbing.

Databricks Apps lets you deploy a web application directly onto the platform — co-located with your data, models, and agents. The platform handles hosting, scaling, and identity so you focus on the application logic.

  • Deploy FastAPI, Streamlit, or any Python/Node app with a single command
  • Credential passthrough — the app runs with the user's identity, not a service account
  • Direct access to SQL Warehouses, Vector Search, Model Serving, and Genie
  • Governed by Unity Catalog — the app only surfaces data the user is allowed to see
  • Scale to zero when idle, auto-scale under load — no infra team needed

Databricks Apps — template selection · deployed app details

Getting started

Your data should work
for your people.

Lakebase, Genie, and Databricks Apps aren't three separate products — they're one platform story. Data that's always current, insights that anyone can access, and AI applications that ship in days.

Step 01 — Data

Identify a Lakebase candidate

Pick one operational data source — CRM, ERP, HRMS — and move it to open format on the lakehouse. Eliminate one pipeline.

Step 02 — Analytics

Create a Genie Space

Point Genie at your highest-value UC table. Seed 5 sample questions. Let stakeholders try it this week — no dashboard needed.

Step 03 — Application

Ship one AI app

Take your most-requested internal tool. Wire it to a Genie Space or agent endpoint. Deploy it as a Databricks App.
