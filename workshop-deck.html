Business Semantics
Metric Views — Business Logic Next to Your Data
Define KPIs once in Unity Catalog. Every downstream tool — Genie, dashboards, SQL, BI — consumes the same governed definitions.
🆓
Native Databricks Feature — No Extra Cost
No licensing fees, no desktop authoring tool, no proprietary format. Metric views are built into the lakehouse.
Without Metric Views
✗ Business logic siloed in each BI tool — Power BI has one hire rate definition, Tableau has another
✗ HR leadership gets different numbers from different reports — trust breaks down
✗ Updating a KPI definition means hunting through dozens of dashboards and SQL scripts
✗ AI tools see raw column names — Genie can't reliably answer "What's our hire rate?"
With hiring_analytics Metric View
✓ One definition, everywhere — HireRate = AVG(hired) defined once, consumed by dashboards, Genie, and raw SQL equally
✓ BI tools connect directly — Power BI, Tableau, and AI/BI dashboards consume the metric view natively, no semantic rebuild
✓ One change propagates everywhere — update the view, every report and Genie answer updates automatically
✓ Genie speaks business language — metric labels let recruiters ask "Hire rate for MBA candidates in NJ?" without knowing schema details
bx4.hrd_2030.hiring_analytics
CREATE OR REPLACE VIEW hiring_analytics
  WITH METRICS LANGUAGE YAML AS $$
  dimensions:
    - EducationLevel, Location, Certifications
  measures:
    - HireRate: AVG(hired)
    - AvgTotalScore: AVG(total_score)
    - TotalCandidates: COUNT(DISTINCT id)
$$

Train, track, and deploy a classification model that predicts candidate hiring outcomes using structured scoring data.
📊

Training Data

10 candidates × 7 feature scores: education, experience, leadership, certification, skills, industry fit, and interview performance.

🔬

3 Model Experiments

LogisticRegression, RandomForest, GradientBoosting compared side-by-side in MLflow UI — best model wins.

📈

Tracked Metrics

Accuracy, AUC-ROC, F1-score, confusion matrix — logged per run automatically via MLflow autolog.

🏛️

UC Model Registry

Best model registered to Unity Catalog as bx4.hrd_2030.hire_right_model with @champion alias for safe promotion.

⚡

Model Serving

Scale-to-zero endpoint deployed for real-time predictions — Agent calls it per candidate request.

Response Agent
Hire Right Agent — AI-Powered Hiring Analysis
A ChatAgent (response_agent pattern) with 4 specialized tools — uses Genie Space for data access instead of SQL UDFs.
🔍
query_genie
Genie Space REST API — natural language → SQL. HR analytics answers grounded in your governed Gold tables.
📄
search_resumes
Vector Search — semantic resume retrieval. Returns top-3 most relevant candidates by embedding similarity.
🤖
predict_hiring_score
ML model endpoint — 0/1 hire prediction + probability score + per-feature breakdown for each candidate.
📧
send_email
Mailgun API — HTML-formatted hiring report sent directly to the hiring manager's inbox.

Response Agent Pattern

MLflow 3.0 for Agents
Observe, Evaluate & Deploy Agents with MLflow
End-to-end lifecycle management for AI agents — from development to production governance.
🔍

Tracing

mlflow.langchain.autolog() captures every LLM call, tool execution, input/output automatically. Full observability without extra instrumentation. Shows in MLflow UI as a trace hierarchy with latency and token counts per span.

📊

Evaluations

LLM-as-judge scoring: Correctness, Safety, RelevanceToQuery, custom HR_Guidelines. Run offline evals before shipping to production. Results tracked per MLflow run — compare eval scores across agent versions.

🚀

Agents.Deploy

agents.deploy() deploys a registered ChatAgent to Model Serving with scale-to-zero. Full UC lineage tracking. One command from a registered model to a live endpoint with streaming support.
